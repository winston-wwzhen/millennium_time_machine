# 彩蛋系统设计

> 大量彩蛋 + 探索发现 = 用户惊喜与留存
>
> 讨论日期：2025-12-26

---

## 一、设计理念

### 核心原则

```
所有可见按钮都可点击 + 大量隐藏彩蛋
            ↓
     用户探索 → 发现彩蛋 → 惊喜感
            ↓
     激发更多探索兴趣 → 分享传播
```

### 目标

| 目标 | 说明 |
|------|------|
| **可探索性** | 每个角落都可能藏着惊喜 |
| **多层次** | 从明显到传说，不同深度的彩蛋 |
| **怀旧感** | 彩蛋本身也符合千禧年代调性 |
| **可传播** | 有些彩蛋值得分享到朋友圈 |

---

## 二、彩蛋层级设计

### 四层彩蛋体系

```
第一层：明显彩蛋（容易发现）→ 15个
    ↓
第二层：隐藏彩蛋（需要探索）→ 40个
    ↓
第三层：深度彩蛋（条件组合）→ 20个
    ↓
第四层：传说彩蛋（极难发现）→ 5个
```

**总计**：80个彩蛋

---

## 三、彩蛋类型

### 按触发方式分类

| 类型 | 触发方式 | 示例 |
|------|---------|------|
| **点击彩蛋** | 连续点击特定目标 | 点小狮子10次跳舞 |
| **时间彩蛋** | 特定时间访问 | 午夜0点进入 |
| **操作彩蛋** | 特定操作序列 | 打开所有图标再关闭 |
| **输入彩蛋** | 输入特殊内容 | 聊天框输入"rosebud" |
| **组合彩蛋** | 多个条件组合 | 特定日期+特定操作 |

### 检测器

```javascript
// egg-detector.js
class EggDetector {
  // 检测点击
  checkClick(target, count) {
    return eggs.find(e =>
      e.trigger.target === target &&
      e.trigger.action === 'click' &&
      e.trigger.count === count
    )
  }

  // 检测时间
  checkTime(hour, minute) {
    return eggs.find(e =>
      e.trigger.type === 'time' &&
      e.trigger.hour === hour
    )
  }

  // 检测序列
  checkSequence(sequence) {
    return eggs.find(e =>
      e.trigger.type === 'sequence' &&
      JSON.stringify(e.trigger.sequence) === JSON.stringify(sequence)
    )
  }

  // 检测输入
  checkInput(input) {
    return eggs.find(e =>
      e.trigger.type === 'input' &&
      e.trigger.keywords.includes(input)
    )
  }

  // 触发彩蛋
  trigger(egg) {
    // 执行彩蛋效果
    executeReward(egg.reward)

    // 记录发现
    discoverEgg(egg.id)

    // 通知用户
    showNotification(egg)
  }

  // 记录发现
  discoverEgg(eggId) {
    user.discoveredEggs.push(eggId)
    saveUserProgress()
  }
}
```

## 十一、设计原则

### 核心原则

| 原则 | 说明 |
|------|------|
| **不影响主流程** | 不影响用户正常使用 |
| **有怀旧感** | 彩蛋本身也有年代感 |
| **有惊喜感** | 发现彩蛋时要有"哇"的感觉 |
| **可传播** | 有些彩蛋值得分享 |
| **适度数量** | 不宜太多，保持神秘感 |
| **持续更新** | 定期添加新彩蛋 |

---

### 彩蛋与功能绑定原则

**重要：每次添加新功能或应用时，必须配套设计相关彩蛋！**

| 新功能类型 | 配套彩蛋示例 |
|-----------|--------------|
| **新增页面** | 页面专属彩蛋（点击、时间、操作） |
| **新增应用** | 应用激活彩蛋 + 使用彩蛋 + 隐藏内容彩蛋 |
| **新增功能** | 功能解锁彩蛋 + 使用过程彩蛋 + 达成彩蛋 |
| **新增AI** | AI发现彩蛋 + 特殊对话彩蛋 + 隐藏模式彩蛋 |
| **新增游戏** | 游戏彩蛋（速通、隐藏结局、特殊操作） |
| **新增主题** | 主题解锁彩蛋 + 主题特效彩蛋 |
| **新增活动** | 活动限定彩蛋 + 参与彩蛋 + 完成彩蛋 |

---

### 彩蛋设计检查清单

每次开发新功能时，检查以下问题：

| 问题 | 说明 |
|------|------|
| ✅ 这个功能有哪些点击点？ | 可以做成点击彩蛋 |
| ✅ 这个功能有什么特殊状态？ | 可以做成状态彩蛋 |
| ✅ 这个功能在特定时间有不同表现吗？ | 可以做成时间彩蛋 |
| ✅ 用户完成这个功能有什么成就？ | 可以做成成就彩蛋 |
| ✅ 这个功能可以和其他功能联动吗？ | 可以做成组合彩蛋 |
