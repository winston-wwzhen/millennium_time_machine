> 讨论日期：2025-12-26

---

## 一、设计理念

### 核心逻辑

```
用户获得"值得分享"的内容
        ↓
    便捷的分享入口
        ↓
    一键生成分享内容
        ↓
    好友看到 → 进入小程序
        ↓
    新用户被吸引 → 也想分享
        ↓
        裂变增长
```

### 设计原则

| 原则 | 说明 |
|------|------|
| **便捷优先** | 最多2步完成分享 |
| **内容为王** | 分享内容本身有趣/有价值 |
| **奖励驱动** | 分享有明确奖励 |
| **社交货币** | 分享能展示用户的"逼格" |
| **不打扰** | 不强制，不频繁弹窗 |

## 八、分享流程设计

### 完整流程

```
用户完成游戏/获得成就
    ↓
弹出分享卡片（可选关闭）
    ↓
用户点击"分享"
    ↓
生成分享图片/卡片
    ↓
用户保存图片或直接转发
    ↓
好友看到 → 点击小程序码 → 进入小程序
    ↓
触发回流奖励 → 双方获得奖励
    ↓
新用户完成游戏 → 也要分享
    ↓
裂变循环
```

### 海报生成

```javascript
// canvas生成海报
async generateSharePoster(result) {
  const ctx = wx.createCanvasContext(500, 800)

  // 背景
  ctx.fillStyle = '#1a1a1a'
  ctx.fillRect(0, 0, 500, 800)

  // 标题
  ctx.fillStyle = '#FF9EC7'
  ctx.font = 'bold 32px Arial'
  ctx.fillText('千禧时光机', 250, 60)

  // 结局
  ctx.fillStyle = '#FFF'
  ctx.font = 'bold 48px Arial'
  ctx.fillText(result.title, 250, 200)

  // 小程序码
  const qrCode = await getQRCode()
  ctx.drawImage(qrCode, 150, 650, 200, 200)

  // 保存
  wx.canvasToTempFilePath({
    canvasId: 'shareCanvas',
    success: (res) => {
      this.shareImagePath = res.tempFilePath
    }
  })
}
```

## 十、数据指标

### 核心指标

| 指标 | 说明 | 目标值 |
|------|------|--------|
| **分享率** | 用户完成游戏后分享的比例 | >15% |
| **回流率** | 分享带来的新用户比例 | >5% |
| **转化率** | 新用户完成注册的比例 | >30% |
| **K因子** | 平均每个用户带来新用户数 | >1.2 |

## 十一、注意事项

### 避免过度打扰

| 避免 | 替代 |
|------|------|
| ❌ 每次都弹窗 | ✅ 可关闭 + 记住选择 |
| ❌ 强制分享 | ✅ 主动分享有奖励 |
| ❌ 频繁引导 | ✅ 适度提示，不过度 |

---

### 合规注意

| 注意 | 说明 |
|------|------|
| **不欺骗** | 分享内容真实，不夸大 |
| **不诱导** | 不诱导用户点击 |
| **不强制** | 用户自愿分享 |
| **不刷屏** | 分享有频率限制 |
