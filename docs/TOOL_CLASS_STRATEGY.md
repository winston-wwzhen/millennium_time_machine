# 工具类定位策略文档

> 个人主体小程序合规化设计方案

> 讨论日期：2025-12-26

---

## 背景

### 个人主体小程序限制

| 限制类型 | 具体影响 |
|---------|---------|
| **社交功能** | ❌ 不能做真社交、朋友圈、关注系统 |
| **聊天功能** | ❌ 只能AI聊天，不能真人社交 |
| **支付功能** | ❌ 不能微信支付 |
| **游戏功能** | ❌ 个人主体游戏类目很难过审 |
| **类目限制** | ❌ 很多类目个人主体选不了 |

### 当前审核定位

**类目**：工具类

**现有策略**：
- AI 模拟社交（不是真社交）
- 小游戏点缀（不是游戏平台）
- Windows 98 怀旧界面（文化展示）

## 核心策略：工具类包装

### 审核话术转换

| 原元素 | 工具类包装 | 审核描述 |
|--------|-----------|---------|
| 游戏玩法 | 历史年表互动体验 | "2005-2025年历史事件回顾工具" |
| 游戏结局 | 人生可能性展示 | "基于历史选择的人生路径模拟器" |
| 属性系统 | 成长维度分析 | "个人成长多维度评估" |
| 重新游戏 | 重新体验 | "探索不同人生路径" |

### 定位原则

```
审核视角：这是一个"历史年表浏览+人生选择模拟"的教育/回忆工具
         ↓
用户体验：玩法依然是人生模拟游戏，但包装更温和
         ↓
过审关键：弱化"游戏"标签，强化"内容/回忆/选择"
```

### 文案调整原则

**重要：文案调整要最小化，确保用户体验不受影响**

| 原则 | 说明 |
|------|------|
| **关键入口调整** | 首屏、主入口的文案必须改，给审核第一印象 |
| **内部体验保留** | 用户进入后，内部文案可适当保留，维持体感 |
| **核心词替换** | "游戏"→"体验"，"玩家"→"用户"，其他可保留 |
| **删除 > 改文案** | 对于明显游戏化功能，直接删除比改文案更干净 |

## 全项目审核风险分析

### 功能模块风险分析

| 功能模块 | 风险等级 | 风险点 | 建议 |

| **网上邻居** | 🟢 低 | 功能模拟，无风险 | 无需调整 |
| **QCIO社交系统** | 🟡 中 | 模仿QQ空间，可能被认定为"社交" | 强化"AI模拟"定位 |
| **AI聊天** | 🟢 低 | AI非真人，相对安全 | 确保文案不强调"社交" |
| **踩一踩** | 🟡 中 | 访客、留言板像社交 | 可考虑弱化或包装 |
| **心情日志** | 🟢 低 | 记录工具，无风险 | 无需调整 |
| **俄罗斯方块** | 🟢 低 | 已删除 | - |
| **星际探索** | 🔴 高 | 明显的游戏功能 | **建议删除**（非核心功能） |
| **火星翻译** | 🟢 低 | 文字转换工具 | 强调"工具"属性 ✅ |
| **如果当时.exe** | 🟡 中 | 人生模拟游戏 | 已讨论，需包装 |
| **非主流相机** | 🟢 低 | 头像生成工具 | 强调"工具"属性 ✅ |
| **等级系统** | 🟡 中 | 经验值、升级像游戏 | 需要包装 |
| **钱包/Q点** | 🟡 中 | 虚拟货币系统 | 需要注意 |

### 高风险细节 - 包装方案

#### 1. 小游戏（星际探索）

**问题**：太明显是游戏

**策略：保留，低调处理**

| 理由 | 说明 |
|------|------|
| 体验完整性 | Windows 98 桌面本就应有小游戏，符合怀旧主题 |
| 用户留存 | 小游戏能增加停留时长和用户粘性 |
| 审核有先例 | 很多工具类小程序有小游戏点缀，只要不是主业 |
| 风险可控 | 审核不通过再删除，调整成本低 |

**低调处理方案**：
- 不放桌面图标，只放开始菜单
- 改名为 "Star.exe"
- 审核话术："展示2005年经典PC软件，怀旧用途"
- 内部文案尽量保留，维持用户体验

**备选方案（如果审核不通过）**：
- 直接删除星际探索小游戏模块

#### 2. QCIO 社交系统

**问题**：好友、群聊、留言板太像社交

**包装方案**：
| 原定位 | 建议定位 |
|--------|---------|
| QCIO社交空间 | "AI互动体验区" |
| 好友列表 | "AI联系人" |
| 群聊 | "AI群组对话" |
| 踩一踩 | "访问体验" |
| 留言板 | "AI留言记录" |

**关键**：所有文案强调"AI"和"体验"，弱化"社交"

#### 3. 等级系统

**问题**：经验值、升级、成就太游戏化

**包装方案**：
| 原定位 | 建议定位 |
|--------|---------|
| 等级 | "活跃度" / "成长值" |
| 经验值 | "互动记录" |
| 升级 | "阶段提升" |
| 成就 | "里程碑" |

#### 4. 钱包/Q点系统

**问题**：虚拟货币可能被认定为付费游戏元素

**包装方案**：
| 原定位 | 建议定位 |
|--------|---------|
| 钱包 | "记录统计" |
| Q点 | "互动次数" |
| 充值 | 完全移除（个人主体不能支付） |

### 文案风险清单

#### 需要避免的关键词

| 类别 | 避免使用 | 替代表述 |
|------|---------|---------|
| 游戏 | 游戏、玩家、通关 | 体验、程序、软件、完成 |
| 社交 | 好友、关注、粉丝 | AI联系人、AI互动 |

- **AI** - 强调非真人互动
- **工具** - 强调实用属性
- **回忆** - 强调文化展示
- **体验** - 替代"游戏"
- **经典** - 替代"游戏"
- **程序** - 替代"游戏"

## 预热与上线策略

### 预热风险评估

| 风险 | 现实 |
|------|------|
| 怕Idea被抄 | Idea不值钱，执行才值钱 |
| 怕功能被抄 | 他们抄得了一时，追不上内容更新 |
| 怕用户被抢 | 怀旧是情感连接，有忠诚度 |
| 怕做不起来 | 先做起来，再想别的 |

### 核心逻辑

```
他们能抄功能，抄不了内容；
抄了内容，抄不了用户；
抄了用户，抄不了品牌。
```

---

### 推荐策略：内容预热 + 快速上线

```
Week 1-2: 发怀旧内容，建立内容IP（公众号、小红书）
Week 3:   社群预热，神秘感营销
Day -3:   统一预告"即将上线"
Day 0:    正式上线，全网爆发
```

---

### 预热内容策略（降低泄露风险）

| 可以展示 | 不要展示 |
|---------|---------|
| 怀旧情感、年代回忆 | 具体功能截图 |
| 非主流文化、QQ空间 | 产品细节 |
| "能回到2005年" | 技术实现方案 |
| 情感共鸣内容 | 数据结构、事件列表 |

**预热内容示例**：
```
✅ "还记得2005年的QQ空间吗？黄钻、踩一踩、火星文..."
❌ "我们的小程序有50个历史事件、100个AI好友"
```

---

### 三种预热方案对比

| 方案 | 优点 | 缺点 | 推荐 |
|------|------|------|------|
| **内容预热** | 零风险，内容即壁垒 | 需要时间积累 | ✅ |
| **私域预热** | 精准、转化高 | 覆盖面小 | ✅ |
| **神秘预热** | 有创意、吸引好奇 | 可能影响理解 | 可选 |

---

### 内容预热方案（推荐）

**不做产品预热，做内容预热**

| 动作 | 说明 |
|------|------|
| 公众号 | 发怀旧文章，建立内容IP |
| 社群 | "怀旧交流群"，不提产品 |
| 小红书 | 怀旧内容，末尾轻描淡写"在做个小程序" |
| 上线前1天 | 统一发布"明天上线" |
